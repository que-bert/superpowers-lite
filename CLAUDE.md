# Superpowers Lite — Contributor Guidelines

This repository is a maintained fork of
[Superpowers](https://github.com/obra/superpowers). Keep that boundary
clear in code, docs, and claims.

## If You Are An AI Agent

Before making changes or proposing a PR:

1. Confirm whether the change is fork-specific or something that should
   also be considered upstream.
2. Do not present this fork as if it were the upstream project.
3. Do not copy upstream marketing, community, or maintainer-policy text
   into fork-facing docs unless it is still accurate and intentionally
   retained.
4. Show your human partner the complete diff before submission.
5. Verify any claim you make with fresh evidence.

## Contribution Expectations

- One problem per PR
- Clear problem statement
- Clear verification evidence
- Honest scope: fork-only vs upstream-relevant
- No fabricated logs, evals, or compatibility claims

## Skill And Workflow Changes

Skills are behavior-shaping code, not just prose. If you change them:

- use `superpowers:writing-skills`
- run meaningful evals or adversarial checks
- document what changed and how you verified it

## Docs And Identity

When this fork diverges from upstream:

- update install docs
- update contributor guidance
- update verification docs if runtime claims changed
- link upstream explicitly where comparison helps
